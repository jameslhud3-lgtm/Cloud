Lesson 1: What is Code?

Code is just instructions written for a computer. That's it.

You know how a recipe tells you step by step how to make a cake? Code is a recipe for a computer. It tells the computer what to do, step by step.

Think of it like this A recipe says: "Mix flour and eggs. Bake at 350 for 30 minutes."

Code says: "Make the background blue. Put a button here. When someone taps it, do this."

The snake game we made earlier? That was code telling the computer: "Draw a green square. Move it when someone taps an arrow. If it hits a wall, game over."

You don't need to write code yourself. You just tell me what you want, and I write the code for you.

Lesson 2: What is a File?

A file is where code gets saved. Just like how you save a photo on your phone, code gets saved in a file.

The snake game? All of it lives inside one file called snake.html

Think of it like this Your phone has photo files (pictures), music files (songs), and video files (movies).

Code files are the same idea -- they're just files that contain instructions instead of photos or music.

The .html part at the end tells the computer: "This file is a webpage. Open it in a web browser."

That's why you could tap that link earlier and see the game -- your phone's browser read the file and followed the instructions inside it.

Lesson 3: What is GitHub?

GitHub is a website where people store their code files. Think of it like Google Drive or iCloud, but for code.

Think of it like this iCloud saves your photos online so you can see them from any device.

GitHub saves your code online so anyone can see it from any device.

When I save your snake game, it goes to GitHub. That's why you could tap that link and play it -- your phone grabbed the file from GitHub and showed it to you.

You have your own space on GitHub. It's like your own folder. The address is:

Your GitHub space github.com/jameslhud3-lgtm/Cloud

Everything we build together gets saved there.

Lesson 4: What is Claude Code?

Claude Code is what you're using right now to talk to me. It's the tool where you type messages and I build things for you.

Think of it like this Imagine you hired a builder to make you a house. You don't need to know how to use a hammer. You just say "I want a blue house with 3 rooms" and the builder does it.

Claude Code is like that. You tell me what you want. I write the code. I save it. I give you a link to see it.

Here's what happens when you ask me to make something:

The steps 1. You say what you want (example: "make me a snake game")
2. I write the code and save it to a file
3. I upload that file to GitHub
4. I give you a link to tap and see it

You never need to touch the code. Just tell me what you want in plain English.

Lesson 7: The Preview Link

Remember how you tapped a link to play the snake game? Let's understand how that works.

After I build something and upload it to GitHub, I'll always give you a tappable link that looks like blue text you can tap.

Think of it like this When someone texts you a YouTube link and you tap it, a video opens.

Same thing here. I give you a link, you tap it, and your creation opens in your browser.

How to save it:

Save it to your home screen 1. Tap the link I give you
2. It opens in your browser
3. Tap the Share button (square with arrow on iPhone, three dots on Android)
4. Tap "Add to Home Screen"
5. Now it's on your phone like an app!

If you ever lose the link, just come back here and ask me: "Give me the link" and I'll give it to you again.
